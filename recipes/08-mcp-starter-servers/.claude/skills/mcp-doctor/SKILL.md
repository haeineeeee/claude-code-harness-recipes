# MCP Doctor — 연동 진단 스킬

## 트리거
사용자가 다음 키워드를 사용할 때 활성화:
- "MCP 진단", "mcp doctor", "연동 상태", "서버 점검"

## 절차

1. **환경변수 점검**: SLACK_BOT_TOKEN, SLACK_TEAM_ID, NOTION_API_KEY, GITHUB_TOKEN 존재 여부 확인
2. **서버 프로세스 확인**: `ps aux | grep mcp` 로 실행 중인 MCP 서버 프로세스 목록
3. **감사 로그 분석**: `.claude/mcp-audit.jsonl` 최근 50건에서 서버별 호출 횟수, 에러율 집계
4. **연결 테스트**: 각 서버의 간단한 읽기 도구 1개씩 호출
   - Slack: 채널 목록 1개 조회
   - Notion: 빈 쿼리 검색 1건
   - GitHub: 레포지토리 목록 1개 조회
5. **리포트 출력**: 서버별 상태, 최근 호출 수, 권장 조치를 표로 정리

## 출력 형식

| 서버 | 환경변수 | 프로세스 | 연결 테스트 | 최근 24h 호출 | 상태 |
|------|----------|----------|-------------|---------------|------|

## 주의
- 토큰 값 자체를 절대 출력하지 않는다 (존재 여부만 확인)
- 연결 테스트 실패 시 구체적인 에러 메시지와 해결 방법 제시
