# /parallel — 병렬 서브에이전트 실행

여러 독립 작업을 병렬 서브에이전트로 동시에 처리합니다.

## 사용법

```
/parallel <작업1> | <작업2> | <작업3>
```

## 동작

1. `|`로 구분된 각 작업을 개별 서브에이전트로 분배합니다.
2. 모든 에이전트를 `run_in_background: true`로 병렬 실행합니다.
3. 각 에이전트 완료 시 결과를 수집하여 종합 보고합니다.

## 예시

```
/parallel src/auth 리팩터링 | src/api 테스트 추가 | docs/ 업데이트
/parallel 컴포넌트A 타입 수정 | 컴포넌트B 타입 수정 | 컴포넌트C 타입 수정
```

## 주의사항

- 각 작업은 서로 독립적이어야 합니다 (같은 파일 수정 금지).
- 의존성이 있는 작업은 `/orchestrate`를 사용하세요.
